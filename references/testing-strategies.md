### CDC Polling Latency and Throughput Formulas

When dealing with Change Data Capture, specifically log-based extraction, understanding the theoretical bounds of latency is crucial for performance tuning.

1. **End-to-End Latency Formula:**
   $$ L_{total} = t_{commit} + t_{wal\_flush} + t_{network1} + t_{extract} + t_{transform} + t_{network2} + t_{kafka\_ack} $$
   Where:
   - $t_{commit}$ = Time for the source database to commit the transaction.
   - $t_{wal\_flush}$ = Time for the DB to write the transaction to the Write-Ahead Log (WAL) or Binlog.
   - $t_{network1}$ = Network latency between the DB instance and the CDC Connector (e.g., Debezium).
   - $t_{extract}$ = Processing time for the connector to parse the binary log format.
   - $t_{transform}$ = Time taken by Single Message Transforms (SMTs) in Kafka Connect.
   - $t_{network2}$ = Network latency between the Connector and the Kafka Brokers.
   - $t_{kafka\_ack}$ = Time for Kafka brokers to acknowledge the message (depends on `acks=all` and `min.insync.replicas`).

2. **Throughput Maximum (Messages Per Second):**
   $$ T_{max} = \frac{B_{network}}{S_{avg\_msg} + S_{overhead}} $$
   Where $B_{network}$ is the network bandwidth bottleneck, $S_{avg\_msg}$ is the average payload size (e.g., JSON vs Avro), and $S_{overhead}$ includes TCP/IP headers, Kafka headers, and Schema Registry metadata.

### CDC Polling vs Log-Based Latency

```ascii
Polling:
DB State [A=1] -> wait 5s -> Poll -> Emit [A=1] -> Latency = 5s
DB State [A=2] -> wait 5s -> Poll -> Emit [A=2] -> Latency = 5s

Log-based (Binlog/WAL):
DB State [A=1] -> Commit -> WAL flush -> CDC read -> Emit -> Latency = 50ms
DB State [A=2] -> Commit -> WAL flush -> CDC read -> Emit -> Latency = 50ms
```

package com.j4flmao.kafka.connect.smt;

import org.apache.kafka.common.config.ConfigDef;
import org.apache.kafka.connect.connector.ConnectRecord;
import org.apache.kafka.connect.transforms.Transformation;
import org.apache.kafka.connect.data.Struct;
import org.apache.kafka.connect.data.Schema;
import java.util.Map;

public class CdcTimestampRouter<R extends ConnectRecord<R>> implements Transformation<R> {
    public static final String OVERVIEW_DOC = "Routes CDC records based on transaction timestamp.";
    
    @Override
    public R apply(R record) {
        if (record.value() == null) {
            return record;
        }
        Struct value = (Struct) record.value();
        Struct source = value.getStruct("source");
        if (source != null) {
            Long ts_ms = source.getInt64("ts_ms");
            if (ts_ms != null) {
                String newTopic = record.topic() + "_" + (ts_ms / 86400000L);
                return record.newRecord(
                    newTopic,
                    record.kafkaPartition(),
                    record.keySchema(),
                    record.key(),
                    record.valueSchema(),
                    record.value(),
                    record.timestamp()
                );
            }
        }
        return record;
    }

    @Override
    public ConfigDef config() { return new ConfigDef(); }

    @Override
    public void close() {}

    @Override
    public void configure(Map<String, ?> configs) {}
}

## 3. Python Data Processing Pipelines\n```python

import json
from confluent_kafka import Consumer, KafkaError

def cdc_processor():
    c = Consumer({
        'bootstrap.servers': 'localhost:9092',
        'group.id': 'cdc-analytics-group',
        'auto.offset.reset': 'earliest',
        'enable.auto.commit': False
    })

    c.subscribe(['myserver.inventory.customers'])

    try:
        while True:
            msg = c.poll(1.0)
            if msg is None:
                continue
            if msg.error():
                if msg.error().code() == KafkaError._PARTITION_EOF:
                    continue
                else:
                    print(msg.error())
                    break
            
            # Parse Debezium CDC envelope
            payload = json.loads(msg.value().decode('utf-8'))
            op_type = payload.get('payload', {}).get('op')
            
            if op_type == 'c':
                print(f"INSERT detected: {payload['payload']['after']}")
            elif op_type == 'u':
                print(f"UPDATE detected: {payload['payload']['before']} -> {payload['payload']['after']}")
            elif op_type == 'd':
                print(f"DELETE detected: {payload['payload']['before']}")
                
            # Manual commit for exactly-once processing approximations
            c.commit(msg)
            
    except KeyboardInterrupt:
        pass
    finally:
        c.close()

if __name__ == '__main__':
    cdc_processor()
