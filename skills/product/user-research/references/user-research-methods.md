# User Research Methods

## Research Types

### Generative Research
| Method | When to Use | Output |
|--------|-------------|--------|
| User Interviews | Early discovery, understanding needs | Themes, insights, quotes |
| Contextual Inquiry | Understanding workflows in context | Task models, pain points |
| Diary Studies | Understanding behavior over time | Longitudinal patterns |
| Focus Groups | Exploring reactions to concepts | Group dynamics, opinions |

### Evaluative Research
| Method | When to Use | Output |
|--------|-------------|--------|
| Usability Testing | Testing existing designs | Task success, issues |
| A/B Testing | Comparing two options | Quantitative preference |
| Card Sorting | Information architecture | Content groupings |
| Tree Testing | Navigation validation | Findability metrics |

### Continuous Research
| Method | When to Use | Output |
|--------|-------------|--------|
| Surveys | Quantitative validation | Statistical data |
| Analytics | Behavioral measurement | Usage patterns |
| Feedback | Ongoing input | Feature requests, bugs |
| CSAT/NPS | Satisfaction tracking | Score trends |

| Method | Participants | Time | Analyst Skill |
|--------|-------------|------|---------------|
| Interview | 10-20 | 2-3 weeks | High |
| Usability Test | 5-10 | 1-2 weeks | Medium |
| Survey | 300+ | 1-2 weeks | Low |

| Diary Study | 10-15 | 2-4 weeks | High |

## Session Structure

### Interview Protocol
1. **Warm-up** (5 min): Introduction, consent, context
2. **Background** (10 min): Current behavior, experience
3. **Deep dive** (30 min): Specific scenarios, pain points, goals
4. **Wrap-up** (5 min): Summary, next steps, thanks

### Moderator Tips
- Ask open-ended questions (tell me about, describe, how did you)
- Follow up with "why" for deeper understanding
- Silence is productive — give time to think
- Avoid leading questions
- Listen more than you talk (80/20 rule)

## Synthesis Approaches

### Affinity Mapping
1. Capture observations on individual notes
2. Group by theme without predetermined categories
3. Name each group with the core insight
4. Identify patterns and relationships
5. Prioritize findings by frequency and severity

### Journey Mapping
- Plot user experience across time
- Identify pain points and opportunities
- Capture emotional highs and lows
- Include touchpoints, channels, and actors
- Map to internal processes and systems
