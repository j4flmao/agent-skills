# Synthesis Frameworks

## Affinity Mapping

### Process
```
1. Transcribe observations onto individual notes
   — One insight per note
   — Include participant ID and context

2. Group notes into clusters
   — Look for patterns and repetition
   — Allow themes to emerge organically
   — Each cluster = 3-15 notes

3. Name each cluster
   — Descriptive title capturing the theme
   — e.g., "Users feel lost during setup"
   — Include the frequency and severity

4. Identify relationships
   — How do themes connect?
   — What are root causes vs symptoms?
   — Create hierarchy of themes
```

### Output Format
```
Theme: Setup is confusing
  - P1: "I didn't know what to do after signing up"
  - P3: "There were too many options"
  - P5: "I almost gave up at step 4"
  Severity: High (affects activation)
  Frequency: 6/8 participants
```

## User Journey Mapping

### Components
```
Phases: Stages the user goes through (Awareness → Consideration → Decision → Onboarding → Usage)
Actions: What the user does at each stage
Touchpoints: Where they interact with the product
Emotions: How they feel (happy, frustrated, confused)
Pain points: What frustrates or blocks them
Opportunities: Where we can improve
```

### Journey Template
```
Phase: {phase name}
User Goal: {what user wants to achieve}
Actions: {list of steps}
Touchpoints: {app, email, website, support}
Emotions: 😊 🤔 😤
Pain Points: {list}
Opportunities: {list}
```

### Mapping Rules
```
Base on research data, not assumptions
Show emotional arc throughout journey
Include pre- and post-product stages
Highlight moments of truth (make or break moments)
Identify cross-channel friction
```

## Persona Creation

### Persona Structure
```
Name: {representative name}
Photo: {diverse, realistic stock photo}
Tagline: {one-line summary of their relationship to product}

Demographics:
  Role: {job title}
  Industry: {industry}
  Company size: {employees}
  Technical level: {low / medium / high}

Goals:
  1. Primary goal related to product
  2. Secondary goal
  3. Professional aspiration

Pain Points:
  1. Biggest frustration
  2. Secondary frustration
  3. What they've tried

Behavior:
  Current solution: {what they use now}
  Purchase behavior: {how they evaluate}
  Usage pattern: {how often and when}

Quote:
  "Representative quote from research"
```

### Persona Guidelines
```
Create 3-5 personas max
Base on patterns from 3+ participants
Include negative personas (who NOT to design for)
Share with team for feedback and alignment
Use in design reviews and prioritization
```

## Insight Documentation

### Insight Format
```
Observation: What was observed (neutral, factual)
Interpretation: What it means (analysis)
  Supporting data: Quotes, metrics, frequencies
Recommendation: What to do about it
  Priority: High / Medium / Low
  Confidence: High / Medium / Low
```

### Priority Matrix
```
High Confidence + High Impact → Immediate action
High Confidence + Low Impact → Quick wins, schedule
Low Confidence + High Impact → Validate with more research
Low Confidence + Low Impact → Monitor, revisit later
```

## Research Report

### Executive Summary
```
Problem researched: [question]
Method: [approach]
Participants: [N]
Timeline: [dates]

Top 3 Insights:
1. [Insight with evidence]
2. [Insight with evidence]
3. [Insight with evidence]

Recommendations:
- [Action] → [Expected outcome]
- [Action] → [Expected outcome]
```

### Report Structure
```
1. Executive Summary (1 page)
2. Methodology (recruitment, protocol, limitations)
3. Key Findings (3-5 themes with evidence)
4. Personas (if applicable)
5. Journey Maps (if applicable)
6. Recommendations (prioritized)
7. Appendix (interview guide, raw data links)
```
