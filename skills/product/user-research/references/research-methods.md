# Research Methods

## Qualitative Methods

### User Interviews
```
Format: 1-on-1, 45-60 min, semi-structured
Sample: 5-8 per segment
Best for: Understanding motivations, pain points, context
Output: Transcripts, highlights, themes
```

#### Interview Types
```
Generative: Explore problem space, discover needs
  — "Tell me about a time when..."
  — "Walk me through how you currently..."

Evaluative: Test concepts or prototypes
  — "What do you think this does?"
  — "How would you use this feature?"

Diagnostic: Understand specific behavior
  — "I noticed you did X, what were you thinking?"
  — "What made you decide to..."
```

### Usability Testing
```
Format: Moderated or unmoderated, task-based
Sample: 5-8 per study
Best for: Identifying usability issues, validating flows
Metrics: Task completion, time on task, error rate, SUS score
```

#### Protocol Structure
```
Introduction (5 min): Consent, context, think-aloud instruction
Warm-up (5 min): Simple task to get comfortable
Core tasks (30 min): 3-5 key tasks, each with success criteria
Debrief (10 min): Overall reaction, follow-up questions
```

### Diary Studies
```
Format: Participants log experiences over time
Duration: 5-14 days
Sample: 8-12 participants
Best for: Understanding long-term behaviors, context
```

## Quantitative Methods

### Surveys
```
Format: Structured questionnaire
Sample: 100+ responses per segment
Best for: Validating findings, measuring sentiment
Common Types:
  - NPS survey (single question, 0-10 scale)
  - SUS (System Usability Scale, 10 questions)
  - Feature preference (ranked choice)
  - CES (Customer Effort Score)
```

### In-App Microsurveys
```
Format: 1-3 questions triggered by behavior
Sample: All users who trigger condition
Best for: Moment-specific feedback
Example: "Did you accomplish what you wanted?" after flow
```

## Method Selection Matrix

### By Product Stage
```
Discovery:
  — User interviews (generative)
  — Competitive analysis
  — Diary studies

Validation:
  — Usability testing (prototype)
  — Concept testing
  — Landing page A/B test

Iteration:
  — A/B testing
  — Usability testing (live)
  — In-app microsurveys

Monitoring:
  — Analytics (quantitative)
  — NPS surveys
  — Support ticket analysis
```

### By Time Available
```
1 Week: Usability testing (5 sessions), survey
2 Weeks: User interviews (8 sessions), diary study
1 Month: Full mixed-methods study
```

## Participant Recruitment

### Screener Criteria
```
Demographics: Role, industry, company size
Behavior: Usage frequency, feature adoption
Attitude: NPS segment, satisfaction level
Exclude: Friends/family, competitors, professional testers
```

### Incentive Guidelines
```
Consumer users: $25-50 gift card (30-60 min)
B2B professionals: $75-150 (30-60 min)
Executives: $200-500 (30-45 min)
Diary studies: $50-100 per day
```
