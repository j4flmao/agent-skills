# Brief Templates

Additional brief templates for different project types and formats.

## Template 1: Lean Canvas Format

Use when validating a new idea quickly without writing a full brief.

```markdown
# Lean Canvas: {Project Name}

## Problem
Top 3 problems the product solves:
1. {Problem 1}
2. {Problem 2}
3. {Problem 3}

Existing alternatives: {How do users solve this today?}

## Solution
Top 3 features that solve the problems:
1. {Feature 1}
2. {Feature 2}
3. {Feature 3}

## Key Metrics
{3-5 numbers that indicate success or failure}

## Unique Value Proposition
{Single, clear, compelling message that states why you are different
and worth paying attention to}

## Unfair Advantage
{Something that cannot be easily copied or bought}

## Channels
{How will you reach your target users?}
- {Channel 1}
- {Channel 2}

## Customer Segments
Target customers:
- {Segment 1}
- {Segment 2}

Early adopters: {Who needs this most urgently?}

## Cost Structure
{What are the fixed and variable costs?}
- {Cost 1}
- {Cost 2}

## Revenue Streams
{How will you make money?}
- {Revenue stream 1}
- {Revenue stream 2}
```

## Template 2: One-Page Pitch Format

Use for internal stakeholder alignment and team communication.

```markdown
# One-Page Brief: {Project Name}

## What We Are Building
{One sentence describing the product.}

## Why Now
{Why is this the right time to build this? Market timing, technology
availability, customer demand signals.}

## Who It Is For
{Primary user persona description.}

## What Problem It Solves
{The core problem in one sentence.}

## How It Works (High-Level)
1. {First step user takes}
2. {Second step}
3. {Third step}
4. {Outcome}

## What Success Looks Like
| Metric | 3-Month Target | 6-Month Target |
|--------|----------------|----------------|
| {Metric 1} | {Number} | {Number} |
| {Metric 2} | {Number} | {Number} |

## What We Are NOT Building
- {Out of scope item 1}
- {Out of scope item 2}
- {Out of scope item 3}

## Key Risks
1. {Risk 1} — {Mitigation}
2. {Risk 2} — {Mitigation}

## Timeline
| Milestone | Target Date |
|-----------|-------------|
| {Milestone 1} | {Date} |
| {Milestone 2} | {Date} |
| {Milestone 3} | {Date} |

## Team
- {Role}: {Name}
- {Role}: {Name}
```

## Template 3: Feature Brief Format

Use when adding a significant feature to an existing product.

```markdown
# Feature Brief: {Feature Name}

## Background
{Existing product context. What does the product do today?}

## Problem
{What specific user problem does this feature solve? Include
supporting evidence: user research, support tickets, analytics data.}

## Proposed Solution
{High-level description of the feature. Focus on WHAT the user can
do, not HOW it works internally.}

## Success Criteria
| Criterion | Measurement |
|-----------|-------------|
| {Criterion 1} | {Measurement method} |
| {Criterion 2} | {Measurement method} |

## User Impact
- {Positive impact on users}
- {Potential negative impact or tradeoff}

## Engineering Impact
- {Impact on existing architecture}
- {Dependencies on other teams/systems}
- {Estimated effort range}

## Dependencies
- {Dependency 1}
- {Dependency 2}

## Rollout Plan
- Phase 1: {Internal/dogfood}
- Phase 2: {Beta/early access}
- Phase 3: {General availability}

## Success Metrics Rollup
| Metric | Baseline | Target |
|--------|----------|--------|
| {Metric} | {Current value} | {Target value} |

## Risks
| Risk | Likelihood | Impact | Mitigation |
|------|------------|--------|------------|
| {Risk} | {H/M/L} | {H/M/L} | {Strategy} |
```

## Template 4: API/Platform Brief Format

Use for platform products, APIs, and developer tools.

```markdown
# Platform Brief: {Product Name}

## Vision Statement
{One sentence describing the platform's purpose and long-term vision.}

## Target Developer Persona
{Description of the target developer: frontend/backend, preferred
languages, team size, use case.}

## Core Capabilities
1. {Capability 1} — {One sentence description}
2. {Capability 2} — {One sentence description}
3. {Capability 3} — {One sentence description}

## API Design Principles
- {Principle 1 (e.g., RESTful resources)}
- {Principle 2 (e.g., Consistent error format)}
- {Principle 3 (e.g., Pagination by default)}

## Authentication Model
{How will developers authenticate? API keys, OAuth, JWT?}

## Rate Limits & Quotas
| Tier | Requests/Second | Monthly Cap | Price |
|------|----------------|-------------|-------|
| Free | {n} | {n} | Free |
| Pro | {n} | {n} | ${n} |
| Enterprise | {n} | Custom | Custom |

## SDK Support (MVP)
- {Language 1}
- {Language 2}
- {Language 3}

## Documentation Requirements
- Quickstart guide
- API reference (OpenAPI 3.0)
- SDK code examples
- Migration guides

## Out of Scope (Now)
- {Feature deferred}
- {Feature deferred}
```
