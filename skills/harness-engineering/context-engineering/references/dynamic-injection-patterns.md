# Dynamic Injection Patterns

## On-Demand Context Loading Architecture

Instead of stuffing static, bloated context into the LLM context window upfront, modern context engineering relies on **On-Demand Context Loading**. This pattern queries and injects context fragments dynamically based on the intermediate outputs, state changes, or queries generated by the LLM.

```
                  ┌──────────────────────┐
                  │      User Query      │
                  └──────────┬───────────┘
                             │
                             ▼
                  ┌──────────────────────┐
                  │   Semantic Router    │
                  └──────────┬───────────┘
                             │
            ┌────────────────┴────────────────┐
            ▼                                 ▼
 ┌──────────────────────┐           ┌──────────────────────┐
 │  Vector Embeddings   │           │ Relational DB State  │
 │  (Qdrant/pgvector)   │           │  (Active Task State) │
 └──────────┬───────────┘           └──────────┬───────────┘
            │                                  │
            └────────────────┬─────────────────┘
                             ▼
                  ┌──────────────────────┐
                  │ Context Aggregator   │
                  │   & Token Squeezer   │
                  └──────────┬───────────┘
                             │ (Targeted Payload)
                             ▼
                  ┌──────────────────────┐
                  │  Frontier LLM (API)  │
                  └──────────┬───────────┘
```

### Advantages of Dynamic Context Loading
1. **Token Cost Reduction**: Only input the context chunks relevant to the current user intent, reducing token counts by up to 80% in high-data environments.
2. **Context Window Safety**: Keeps the token usage well below model limits (e.g., GPT-4o or Claude 3.5 Sonnet), preventing truncation errors and ensuring the model sees the entire requested task.
3. **Improved Accuracy**: Reduces distractions in the prompt context by filtering out low-relevance noise, effectively mitigating hallucination risks by providing only high-fidelity chunks.
4. **Modularity**: The system can be swapped from a vector database (RAG) to a live SQL database or a web search tool without modifying the core LLM orchestration logic.

## Vector Search Database Schemas

To retrieve context blocks based on semantic overlap, we use dedicated vector data stores.

### PostgreSQL with pgvector Schema

The following schema defines a production PostgreSQL layout supporting high-dimensional embeddings and dynamic document relationships.

```sql
-- Enable the vector extension
CREATE EXTENSION IF NOT EXISTS vector;

-- Create context registry table
CREATE TABLE IF NOT EXISTS context_registry (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    corpus_name VARCHAR(128) NOT NULL,
    document_chunk TEXT NOT NULL,
    embedding VECTOR(1536) NOT NULL, -- Match size to text-embedding-3-small or OpenAI ada-002
    token_count INT NOT NULL,
    metadata JSONB NOT NULL,
    created_at TIMESTAMP WITH TIME ZONE DEFAULT CURRENT_TIMESTAMP
);

-- Create an HNSW index for high-speed approximate nearest neighbor search
CREATE INDEX ON context_registry 
USING hnsw (embedding vector_cosine_ops)
WITH (m = 16, ef_construction = 64);
```

### Detailed Index Parameter Selection for pgvector
- **m**: Max connections per node in the graph (default: 16). Higher values increase accuracy on high-dimensional vectors but raise build time and memory usage.
- **ef_construction**: Size of the dynamic candidate list for index building (default: 64). Larger values yield more accurate search performance at the cost of indexing speed.

### Performance Tuning Queries (EXPLAIN ANALYZE)
To ensure the PostgreSQL planner uses index scans instead of full sequential scans, tune your SQL parameters:

```sql
-- Force index usage for verification
SET LOCAL enable_seqscan = off;

-- Perform semantic lookup
EXPLAIN ANALYZE
SELECT id, document_chunk, 1 - (embedding <=> '[0.05, 0.12, ..., 0.91]') AS similarity
FROM context_registry
WHERE corpus_name = 'api-docs'
ORDER BY embedding <=> '[0.05, 0.12, ..., 0.91]'
LIMIT 5;
```

## Mathematical Formulation: Similarity & Diversity

Retrieval algorithms often suffer from retrieving duplicate or highly similar chunks. To solve this, we use **Maximal Marginal Relevance (MMR)** to balance relevance and variety:

$$\text{MMR}(Q, R, S) = \arg\max_{D_i \in R \setminus S} \left[ \lambda \cdot \text{Sim}(D_i, Q) - (1 - \lambda) \cdot \max_{D_j \in S} \text{Sim}(D_i, D_j) \right]$$

where:
* $Q$ is the query vector.
* $R$ is the set of retrieved candidate document vectors.
* $S$ is the set of already selected document vectors in the context.
* $\lambda \in [0, 1]$ balances pure relevance ($\lambda = 1$) and pure diversity ($\lambda = 0$).
* $\text{Sim}(u, v)$ is the cosine similarity:

$$\text{Sim}(u, v) = \frac{u \cdot v}{\|u\| \|v\|}$$

## Step-by-Step Injection Pipeline Flow

The dynamic context compiler processes queries according to these actions:

1. **Accept query input** from the user, normalizing the text.
2. **Convert text query into query vector** using a pre-trained embeddings model (e.g., Cohere, OpenAI, or HuggingFace local models).
3. **Execute semantic search** on pgvector / Qdrant to retrieve candidate documents based on the cosine distance threshold.
4. **Iteratively evaluate candidates** against the active tokens cache using the MMR algorithm.
5. **Format template string** with matches that fit the strict budget of the target LLM.
6. **Return prompt payload** for API routing to the chosen frontier LLM.

## Python Dynamic Context Injector

The following production-grade script connects to a simulated vector source, computes similarity, formats templates dynamically, and manages token quotas safely, featuring standard unit test validation.

```python
import numpy as np
import sys
import unittest
from typing import List, Dict, Any, Optional
import tiktoken

class VectorSourceMock:
    """Mock Vector Database Source containing dummy documentation vectors."""
    def __init__(self):
        # Create mock 4D vectors for simplicity
        self.documents = [
            {
                "id": 1,
                "text": "OAuth 2.0 uses bearer tokens to grant secure API access.",
                "vector": np.array([0.9, 0.1, 0.2, 0.0])
            },
            {
                "id": 2,
                "text": "DynamoDB scales horizontally using partition keys.",
                "vector": np.array([0.1, 0.85, 0.0, 0.1])
            },
            {
                "id": 3,
                "text": "JWT tokens contain encoded claims signed with secret keys.",
                "vector": np.array([0.88, 0.15, 0.1, 0.05])
            },
            {
                "id": 4,
                "text": "EKS manages Kubernetes worker nodes automatically.",
                "vector": np.array([0.05, 0.1, 0.9, 0.05])
            }
        ]
        print(f"[DEBUG] Mock Vector DB initialized with {len(self.documents)} records.", file=sys.stderr)

    def query(self, query_vector: np.ndarray, top_k: int = 2) -> List[Dict[str, Any]]:
        results = []
        for doc in self.documents:
            # Cosine similarity calculation
            q_norm = np.linalg.norm(query_vector)
            doc_norm = np.linalg.norm(doc["vector"])
            if q_norm == 0 or doc_norm == 0:
                sim = 0.0
            else:
                sim = np.dot(query_vector, doc["vector"]) / (q_norm * doc_norm)
            
            results.append((doc, sim))
        
        results.sort(key=lambda x: x[1], reverse=True)
        return [{"document": r[0]["text"], "similarity": r[1]} for r in results[:top_k]]

class DynamicContextInjector:
    """
    Orchestrates dynamic retrieval and injection of contexts into prompt systems.
    """
    def __init__(self, db: VectorSourceMock, model_name: str = "gpt-4", max_tokens: int = 150):
        self.db = db
        try:
            self.encoder = tiktoken.encoding_for_model(model_name)
        except Exception:
            self.encoder = tiktoken.get_encoding("cl100k_base")
        self.max_tokens = max_tokens
        print(f"[DEBUG] DynamicContextInjector bound to model '{model_name}' (quota={max_tokens} tokens).", file=sys.stderr)

    def assemble_prompt(self, user_query: str, query_vector: np.ndarray, base_template: str) -> str:
        """
        Dynamically fetches documents, checks token limits, and injects context into the template.
        """
        raw_candidates = self.db.query(query_vector, top_k=3)
        
        injected_contexts = []
        current_token_count = len(self.encoder.encode(base_template.format(context="", query=user_query)))
        
        for idx, cand in enumerate(raw_candidates):
            doc_text = f"- {cand['document']} (Similarity: {cand['similarity']:.2f})\n"
            doc_tokens = len(self.encoder.encode(doc_text))
            
            if current_token_count + doc_tokens <= self.max_tokens:
                injected_contexts.append(doc_text)
                current_token_count += doc_tokens
            else:
                break
                
        context_str = "".join(injected_contexts) if injected_contexts else "No context found.\n"
        return base_template.format(context=context_str, query=user_query)

class TestDynamicContextInjector(unittest.TestCase):
    """Unit tests verifying injection algorithms."""
    
    def test_mock_query(self):
        db = VectorSourceMock()
        results = db.query(np.array([0.9, 0.1, 0.2, 0.0]), top_k=1)
        self.assertEqual(len(results), 1)
        self.assertTrue("OAuth" in results[0]["document"])

if __name__ == "__main__":
    if len(sys.argv) > 1 and sys.argv[1] == "test":
        unittest.main(argv=[sys.argv[0]])
    else:
        db = VectorSourceMock()
        injector = DynamicContextInjector(db=db, max_tokens=150)
        auth_query_vector = np.array([0.95, 0.05, 0.05, 0.0])
        
        template = (
            "System: Use only the context below to answer.\n"
            "Context:\n{context}"
            "User: {query}\n"
            "Assistant:"
        )
        
        final_prompt = injector.assemble_prompt(
            user_query="Explain API security keys.",
            query_vector=auth_query_vector,
            base_template=template
        )
        print("--- Assemble Output ---")
        print(final_prompt)
```
