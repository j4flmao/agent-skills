# Sprint Retrospective and Velocity Tracking

## Algorithms and formulations
Velocity = Total Story Points Completed / Number of Sprints
Capacity = Team Members * Days * Focus Factor

## Data schemas (JSON/YAML)
```json
{
  "sprint": 12,
  "velocity": 42,
  "retro": {
    "went_well": ["communication"],
    "needs_improvement": ["testing"]
  }
}
```

## Code examples (Python, TypeScript)
```python
def calc_velocity(points: int, sprints: int) -> float:
    if sprints == 0:
        return 0.0
    return points / sprints
```
```typescript
function calculateCapacity(members: number, days: number): number {
    return members * days * 0.8; // 80% focus factor
}
```

## Configuration templates
```yaml
retro:
  enabled: true
  format: "Start, Stop, Continue"
velocity_tracking:
  active: true
  moving_average_sprints: 3
```

## Decision matrices
```
[Retro Feedback] -> [Identify Themes] -> [Action Items]
```

$$ V = \frac{\sum_{i=1}^{n} SP_i}{n} $$

## Best practices and anti-patterns

Anti-pattern: Using velocity to compare different teams.
