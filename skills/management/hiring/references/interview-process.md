# Interview Process Design

## Overview
A structured interview pipeline ensures consistent, bias-resistant candidate evaluation from application to offer. Each stage has a clear purpose, format, and decision gating.

## Pipeline Stages

### Stage 1: Application & Screening
Resume reviewed against 5-7 must-have criteria defined before posting. Auto-reject: missing critical requirements. Phone screen (30 min): recruiter or hiring manager — motivation, availability, salary expectations, basic role alignment. Go/no-go gate.

### Stage 2: Technical Assessment
Choose ONE format per role. Take-home: practical, real-world task (4-8 hours, compensated for senior roles). Asynchronous coding: HackerRank, CodeSignal — algorithmic or domain-specific. Pairing session (45 min): live coding with interviewer — preferred for accurate signal.

### Stage 3: Onsite
Typically 3-4 sessions with breaks. Each session is scored independently — no shared rubrics mid-process.
- Coding (45-60 min): algorithm or domain problem, rubric scored
- System Design (45-60 min): architecture problem, rubric scored
- Behavioral (45 min): STAR questions aligned to company values
- Optional: debugging session, API design, or team presentation

### Stage 4: Debrief
Schedule within 24 hours of onsite completion. Attendees: all interviewers, hiring manager, recruiter. Format: each interviewer presents scores silently first (written), then shares verbally. Discrepancy resolution: discuss signals, not personalities. Decision: hire / no-hire / leaning-no (discuss next steps like additional interview or feedback loop).

### Stage 5: Offer
Compile offer with: base salary range (determined before interview), equity/stocks, signing bonus if applicable, start date, shadowing/onboarding plan. Reference checks: 2-3 professional references, focus on collaboration, growth, and impact. Offer expiration: 7-10 business days.

## Candidate Experience
- Send schedule and interviewer names 24h before onsite
- Provide clear expectations for each session (format, duration, tools allowed)
- Debrief feedback to candidate within 48h of debrief (rejections: 24h)
- Every candidate receives a rejection email with actionable feedback (optional for high-volume)

## Bias Mitigation
- Structured interviews: same questions, same rubric for every candidate
- Panel diversity: ensure interviewers reflect diverse perspectives
- Score first, discuss second: prevents anchoring and groupthink
- Blind resume review: remove name, photo, university, years of experience for initial screening
- Decision criteria defined before interviews begin

## Key Points
- Time-to-fill target: 30-45 days from posting to offer acceptance
- Candidate drop-off: most common at the take-home stage (>50% dropout rate)
- Structured interviews predict job performance 2-3x better than unstructured interviews
- Every interviewer must complete interview training before conducting interviews
- Collect candidate experience feedback (NPS) after every onsite
