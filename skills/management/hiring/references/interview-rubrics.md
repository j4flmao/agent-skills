# Interview Rubrics

## Overview
Rubrics provide the objective framework for candidate evaluation. Each dimension is scored 1-4 with behavioral anchors. The rubric is the same for every candidate — never adjusted per person.

## Scoring Scale
| Score | Label | Definition |
|---|---|---|
| 1 | Strong No | Below expectations for level — clear gap |
| 2 | No | Meets some expectations but not enough to advance |
| 3 | Yes | Meets expectations — strong enough to hire |
| 4 | Strong Yes | Exceeds expectations — exceptional candidate |

## Coding Rubric

| Dimension | 1 (Strong No) | 2 (No) | 3 (Yes) | 4 (Strong Yes) |
|---|---|---|---|---|
| Problem-solving | No clear approach, jumps to code | Basic approach, needs hints | Systematic approach, explores tradeoffs | Elegant solution, optimization discussion |
| Communication | Can't explain reasoning | Explains with prompting | Clear explanation throughout | Teaches the interviewer, asks clarifying questions |
| Code quality | Unreadable, no structure | Functional but messy | Clean, well-structured, idiomatic | Production-quality, considers edge cases |
| Testing | No testing consideration | Basic happy path | Covers edge cases, error states | TDD approach, testability built in |

## System Design Rubric

| Dimension | 1 (Strong No) | 2 (No) | 3 (Yes) | 4 (Strong Yes) |
|---|---|---|---|---|
| Requirements | Doesn't ask, guesses scope | Basic clarification | Systematic requirements gathering | Challenges assumptions, prioritizes |
| Architecture | No coherent design | Simple but functional | Well-structured with clear components | Distributed, fault-tolerant, evolvable |
| Scalability | No consideration | Mentions scaling but vague | Concrete scaling strategy (sharding, caching, CDN) | Quantitative scalability with tradeoff data |
| Tradeoffs | No tradeoff awareness | Mentions one tradeoff | Multiple tradeoffs with rationale | Quantitative comparison of alternatives |

## Behavioral Rubric

| Dimension | 1 (Strong No) | 2 (No) | 3 (Yes) | 4 (Strong Yes) |
|---|---|---|---|---|
| Collaboration | Blames others, no teamwork examples | Works alone but can collaborate | Strong team player, unblocks others | Elevates team, mentors, creates inclusion |
| Ownership | Blames circumstances | Takes ownership when asked | Proactively owns outcomes | Takes ownership beyond scope, drives results |
| Growth mindset | Resists feedback | Accepts feedback | Actively seeks feedback and improves | Teaches others, creates learning culture |

## Calibration
- Every 3-6 months, calibrate rubrics across the interviewer team
- Use example candidates (recordings or mock interviews) to align scores
- Flag interviewers who consistently score above/below/outside the team average
- Rubric calibration is a continuous process — update as you learn what signals matter

## Key Points
- Rubric dimensions are non-negotiable — score every dimension for every candidate
- Behavioral anchors prevent arbitrary scoring — use the descriptions in each cell
- A score of 4 should be rare — it means genuinely exceptional, not "good"
- Calibration sessions prevent drift — without them, rubrics lose their value over time
