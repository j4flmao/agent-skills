# Vendor Selection Process

## Introduction

Vendor selection is a structured process to identify, evaluate, and select third-party providers that best meet organizational requirements. A rigorous selection process reduces risk, ensures value for money, and establishes the foundation for a successful vendor relationship.

## RFI / RFP Process

### Request for Information (RFI)
**Purpose**: Gather market intelligence, identify qualified vendors, narrow the field.

| Phase | Activity | Duration | Output |
|-------|----------|----------|--------|
| Preparation | Define scope, write RFI questions, identify distribution list | 1-2 weeks | RFI document |
| Distribution | Send to 5-15 vendors via procurement portal or email | 1 day | Distribution confirmation |
| Response Period | Vendors prepare responses, conduct Q&A | 2-3 weeks | Vendor responses |
| Evaluation | Review responses, score, shortlist 3-5 vendors | 1-2 weeks | Evaluation matrix, shortlist |
| Shortlist Decision | Select vendors to receive RFP | 1 week | Shortlist notification |

### Request for Proposal (RFP)
**Purpose**: Obtain detailed proposals including pricing, approach, and deliverables.

| Phase | Activity | Duration | Output |
|-------|----------|----------|--------|
| Preparation | Write RFP with detailed requirements, pricing templates, evaluation criteria | 2-4 weeks | RFP document |
| Bidders Conference | Present RFP, answer questions, clarify requirements | 1 session | Meeting minutes, Q&A addendum |
| Response Period | Vendors prepare proposals, site visits if needed | 3-6 weeks | Vendor proposals |
| Evaluation | Score proposals, conduct demos, check references | 2-4 weeks | Scoring matrix, demo evaluations |
| Best and Final Offer (BAFO) | Request final pricing from top 1-2 vendors | 1-2 weeks | BAFO responses |
| Selection | Final decision, negotiation preparation | 1 week | Selection memo |

## Evaluation Criteria

### Weighted Scoring Framework

| Category | Weight | Sub-criteria |
|----------|--------|-------------|
| Technical Capability | 30% | Feature fit, integration capability, scalability, architecture alignment |
| Commercial Terms | 25% | Total cost of ownership, pricing model, contract terms, discount structure |
| Security and Compliance | 20% | Security certifications, data protection, compliance with regulations, incident response |
| Support and Service | 15% | Support model, SLAs, account management, training, documentation |
| Company Viability | 10% | Financial stability, market position, customer base, roadmap |

### Technical Evaluation Criteria

| Criterion | Weight | Scoring Guide (1-5) |
|-----------|--------|---------------------|
| Functional coverage | 25% | 1 = <50% requirements met, 5 = >95% requirements met |
| Integration capabilities | 20% | 1 = Custom integration only, 5 = Native APIs and pre-built connectors |
| Performance and scalability | 20% | 1 = Manual scaling, 5 = Auto-scaling with proven SLAs |
| Architecture and standards | 15% | 1 = Proprietary, 5 = Open standards, cloud-native |
| User experience | 10% | 1 = Poor UX, 5 = Intuitive, accessible, configurable |
| Implementation methodology | 10% | 1 = No methodology, 5 = Agile with proven track record |

### Security Evaluation Criteria

| Criterion | Weight | Minimum Threshold |
|-----------|--------|------------------|
| SOC 2 Type II report | 25% | Required for all vendors |
| ISO 27001 certification | 20% | Required for critical vendors |
| Data encryption (at rest/in transit) | 15% | AES-256 / TLS 1.2+ |
| Penetration testing frequency | 15% | At least annually |
| Incident response capability | 10% | Documented IR plan with SLA |
| Vendor access controls | 10% | MFA, least privilege, access reviews |
| Business continuity / DR | 5% | RTO/RPO documented and tested |

## Scoring Methodology

### Scoring Scale
| Score | Definition | Description |
|-------|------------|-------------|
| 5 | Exceptional | Exceeds requirements, provides additional value |
| 4 | Strong | Fully meets requirements with minor gaps |
| 3 | Adequate | Meets core requirements, some gaps |
| 2 | Weak | Partially meets requirements with significant gaps |
| 1 | Unsatisfactory | Does not meet requirements |

### Calculation
```
Weighted Score = Sum (Criteria Weight x Criteria Score)
Maximum Possible Score = Sum (Criteria Weight x 5)
Percentage Score = (Weighted Score / Maximum Possible Score) x 100
```

### Minimum Thresholds
- Technical: >= 70% of maximum
- Commercial: >= 60% of maximum
- Security: >= 80% of maximum (no individual criterion below 3)
- Overall: >= 70% to proceed to BAFO

## Vendor Demonstrations

### Demo Structure
| Segment | Duration | Focus |
|---------|----------|-------|
| Vendor overview | 15 min | Company background, roadmap |
| Solution demonstration | 60 min | Pre-defined scenarios and use cases |
| Technical Q&A | 30 min | Architecture, integration, customization |
| Commercial Q&A | 15 min | Pricing, licensing, support models |

### Demo Scoring Areas
- Requirement coverage (scenarios demonstrated)
- Solution depth (advanced features, edge cases)
- User experience (intuitiveness, efficiency)
- Technical capability (performance, scalability)
- Team expertise (product knowledge, response quality)

## Reference Checks

### Reference Check Questions
1. How long have you used the vendor and what is the scope?
2. How was the implementation experience (timeline, support, issues)?
3. How does the vendor handle support and issue escalation?
4. Has the vendor met their SLA commitments?
5. How has the product evolved since implementation?
6. What are the biggest strengths and weaknesses?
7. Would you select this vendor again?
8. Describe the vendor's responsiveness during critical incidents.

### Reference Verification
- Confirm reference is current customer (not former)
- Speak with technical and business contacts
- Ask about hidden costs not in proposal
- Verify security and compliance claims
- Check product roadmap commitments

## Selection Decision

### Decision Documentation
- CEO (Chief Evaluation Officer) recommendation with rationale
- Evaluation matrix with scores and comments
- Risk assessment summary for recommended vendor
- Total cost of ownership (TCO) comparison
- Negotiation mandate and authority level
- Implementation resource estimate
- Contract term recommendation
