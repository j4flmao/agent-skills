# Reverse ETL Monitoring
Reverse ETL syncs require monitoring for data accuracy, timeliness, and completeness to ensure operational tool data is trustworthy.

## Sync Monitoring
- Track sync success rate by destination
- Monitor record counts and data volume transferred
- Alert on sync failures and partial success
- Measure sync duration and latency trends
- Log errors with detailed context for debugging

## Data Quality Verification
- Validate record counts match source expectations
- Test field-level accuracy with sample checks
- Monitor schema compliance for destination APIs
- Track null rates and default value usage
- Verify identity resolution correctness

## Key Points
- Monitor sync health including success rate and latency
- Verify data quality and completeness post-sync
- Alert on sync failures with detailed error context
- Track operational tool data quality metrics
- Implement sync observability dashboards
