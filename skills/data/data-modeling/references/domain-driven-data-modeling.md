# Domain-Driven Data Modeling

## DDD for Data
Domain-driven design principles applied to data modeling create models that align with business domains and enable autonomous teams.

## Bounded Contexts
- Each domain has its own data model and vocabulary
- Contexts communicate through well-defined interfaces
- Aggregate boundaries ensure data consistency
- Event storming reveals domain events and data flows

## Ubiquitous Language
- Use business terminology in model names
- Align data definitions with team understanding
- Document domain terms in a glossary
- Ensure models reflect business processes

## Key Points
- Apply DDD principles to data model design
- Define clear bounded contexts for autonomous teams
- Use ubiquitous language aligned with business terminology
- Apply event storming to discover domain events
- Design aggregates around business transactions
