# Lineage for Governance
Data lineage supports governance, compliance, and impact analysis by tracing data from source to consumption.

## Compliance Use Cases
- Demonstrate data provenance for regulatory audits
- Track PII and sensitive data through transformations
- Verify data handling compliance with policies
- Support GDPR right-to-explanation requirements
- Document data flows for certification

## Impact Analysis
- Identify downstream consumers of changed datasets
- Assess blast radius of schema modifications
- Plan migration timelines with dependency understanding
- Communicate impacts to affected teams
- Validate no unexpected side effects from changes

## Key Points
- Use lineage for compliance and audit documentation
- Implement impact analysis before schema or pipeline changes
- Track sensitive data through transformations
- Communicate downstream impacts to stakeholders
- Maintain lineage as part of data governance program
