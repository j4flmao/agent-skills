# Data Privacy Compliance

## Regulatory Frameworks
| Regulation | Scope | Key Requirements |
|------------|-------|------------------|
| GDPR | EU personal data | Consent, right to deletion, breach notification (72h) |
| CCPA/CPRA | California residents | Right to know, right to opt-out, right to delete |
| HIPAA | US healthcare | PHI protection, BAAs, audit controls |
| PCI DSS | Payment card data | Encrypt at rest/transit, access control, logging |
| LGPD | Brazil personal data | Similar to GDPR |

## Data Classification Levels
| Level | Label | Examples | Controls |
|-------|-------|----------|----------|
| 1 | Public | Marketing materials, public docs | No restrictions |
| 2 | Internal | Emails, internal wikis | Access control |
| 3 | Confidential | Financial data, employee records | Encryption + access control |
| 4 | Restricted | PII, PHI, payment data | Encryption + access control + audit + masking |

## Privacy by Design Principles
- Data minimization: collect only what's needed
- Purpose limitation: use data only for stated purpose
- Storage limitation: delete when no longer needed
- Transparency: clear privacy notices
- User rights: access, rectification, erasure, portability
