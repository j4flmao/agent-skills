# Incident Response Lifecycle

## NIST 800-61 Framework

### Phase 1: Preparation
- Develop IR plan and procedures
- Establish incident response team
- Deploy prevention and detection tools
- Conduct training and exercises
- Establish communication channels

### Phase 2: Detection & Analysis
- Monitor security alerts
- Validate and triage incidents
- Gather evidence (logs, memory, disk)
- Determine scope and impact
- Document findings

### Phase 3: Containment, Eradication & Recovery
- Short-term containment (isolate host, disable account)
- Long-term containment (apply patches, update rules)
- Eradication (remove malware, close vulnerabilities)
- Recovery (restore from backup, return to production)
- Verify recovery

### Phase 4: Post-Incident Activity
- Lessons learned meeting
- Update incident documentation
- Implement improvements
- Update runbooks and detection rules
- Legal and compliance reporting

## Key Points
- IR lifecycle: Preparation → Detection & Analysis → Containment → Eradication → Recovery → Post-Incident
- Document every action during incident response for post-incident review
- Preserve evidence chain of custody
- Communication plan includes internal and external stakeholders
- Post-incident reviews drive continuous improvement
- Update runbooks and detection rules based on lessons learned
- NIST 800-61 is the standard framework for incident response
