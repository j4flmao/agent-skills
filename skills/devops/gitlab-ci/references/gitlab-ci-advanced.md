# GitLab CI Advanced Topics

## Introduction
Advanced GitLab CI covers parent-child pipelines, multi-project pipelines, CI/CD for monorepos, custom runner auto-scaling, and CI security features.

## Parent-Child Pipelines
Dynamic child pipelines generated from artifacts or code. Parent pipeline triggers child pipelines from YAML or Ruby code. Child pipelines inherit variables from parent. Use for: microservice-based apps, conditional job execution. Key: child pipeline YAML generated by parent job.

## Multi-Project Pipelines
Cross-project CI/CD for microservices. Trigger downstream project pipeline when upstream finishes. Pass variables and artifacts between projects. Status coordination: wait for downstream pipeline status. Use for deployment orchestration across service boundaries.

## CI/CD for Monorepos
Rules: change detection with `changes:` paths. Only build/test affected components. Use include with `rules:changes` for conditional includes. Caching per component for faster builds. Pipeline efficiency: skip unchanged projects.

## Custom Runner Auto-Scaling
Docker Machine executor for AWS, GCP, Azure auto-scaling. Kubernetes executor for dynamic runner pods. Docker Autoscaler (replacement for Docker Machine). Runner scale out/in based on job queue depth. Idle runner count configuration for latency vs cost.

## CI Security
CI job token scope: only required projects. Protected variables scoped to protected branches. Dast, sast, secret detection, container scanning in CI. Dependency scanning with license compliance. Policy-as-code with Scan Result Policies.

## Pipeline Efficiency
DAG (Directed Acyclic Graph): job dependencies with needs keyword. Parallel matrix jobs for multi-version testing. CI/CD cache: keyed by branch, file, or global. Interruptible jobs: auto-cancel redundant pipelines. Resource groups for serial jobs. Job timeout to prevent runaway pipelines.
