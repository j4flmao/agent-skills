# Automatic Changelog Tools

## Tool Comparison

| Tool | Language | Config | Output | CI-Friendly |
|------|----------|--------|--------|-------------|
| git-cliff | Rust (binary) | TOML | Markdown, HTML, custom templates | Yes |
| standard-version | Node.js | package.json | Markdown | Yes |
| semantic-release | Node.js | RC file, config | Markdown + npm/GitHub release | Yes |
| changelogithub | Node.js | Minimal | Markdown from GitHub releases | Yes |
| auto-changelog | Node.js | package.json | Markdown | Yes |
| keep-a-changelog | Python | TOML/Python | Markdown | Yes |
| gren (GitHub Release Notes) | Node.js | config file | Markdown | Yes |

## git-cliff

The fastest and most configurable option. Written in Rust, single binary, no runtime dependencies.

```bash
# Install
cargo install git-cliff

# Generate changelog
git-cliff -o CHANGELOG.md

# Generate for unreleased changes
git-cliff --unreleased -o CHANGELOG.md

# Generate for a specific tag range
git-cliff --tag v1.0.0..v2.0.0
```

### Configuration (cliff.toml)

```toml
[changelog]
header = "# Changelog\n"
body = """
{% for group, commits in commits | group_by(attribute="group") %}
### {{ group | upper_first }}
{% for commit in commits %}
- {{ commit.message | upper_first }}{% if commit.breaking %} (breaking){% endif %}{% if commit.scope %} ({{ commit.scope }}){% endif %}
{%- endfor %}
{% endfor %}
"""
footer = "---\n_Generated by git-cliff_"
template = """
{% for version in versions %}
## [{{ version.version }}] - {{ version.timestamp | date(format="%Y-%m-%d") }}
{% if version.previous %}
[{{ version.version }}]: https://github.com/org/repo/compare/{{ version.previous }}...{{ version.version }}
{% endif %}
{{ version.changelog }}
{% endfor %}
"""

[git]
conventional_commits = true
filter_unconventional = true
split_commits = true

[git.conventional_commits]
# Types that go in the changelog
types = ["feat", "fix", "refactor", "perf", "revert"]
```

## standard-version

```bash
# Install
npm install -D standard-version

# Add to package.json
"scripts": {
  "release": "standard-version",
  "release:minor": "standard-version --release-as minor",
  "release:major": "standard-version --release-as major",
  "release:dry": "standard-version --dry-run"
}

# Run
npm run release
```

### Version bumping
- `fix` commits → PATCH bump
- `feat` commits → MINOR bump
- Breaking changes → MAJOR bump

## semantic-release

Full release pipeline: analyzes commits, bumps version, generates changelog, publishes to npm/GitHub.

```bash
# Install
npm install -D semantic-release @semantic-release/git @semantic-release/changelog

# .releaserc.json
{
  "branches": ["main"],
  "plugins": [
    "@semantic-release/commit-analyzer",
    "@semantic-release/release-notes-generator",
    "@semantic-release/changelog",
    "@semantic-release/npm",
    "@semantic-release/git",
    "@semantic-release/github"
  ]
}
```

### GitHub Actions workflow

```yaml
name: Release
on:
  push:
    branches: [main]
jobs:
  release:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-node@v4
      - run: npm ci
      - run: npx semantic-release
        env:
          GITHUB_TOKEN: ${{ secrets.GITHUB_TOKEN }}
          NPM_TOKEN: ${{ secrets.NPM_TOKEN }}
```

## changelogithub

Lightweight tool that generates changelogs from GitHub releases.

```bash
# Generate from GitHub releases
npx changelogithub
```

Best for projects already managing releases through GitHub's UI. Generates markdown from PR titles, labels, and milestones.

## Changelog Format Standards

### Keep a Changelog

```markdown
# Changelog

## [1.2.0] - 2026-05-14

### Added
- New feature description (#123)

### Changed
- Behavior changes in existing features

### Fixed
- Bug fix descriptions

### Removed
- Removed features

### ⚠️ BREAKING
- Breaking change with migration notes
```

## CI Integration

### GitHub Actions (git-cliff)

```yaml
name: Generate Changelog
on:
  push:
    tags: ['v*']
jobs:
  changelog:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
        with:
          fetch-depth: 0
      - uses: actions-rs/toolchain@v1
        with: { toolchain: stable }
      - run: cargo install git-cliff
      - run: git-cliff -o CHANGELOG.md
      - run: |
          git config user.name github-actions
          git config user.email github-actions@github.com
          git add CHANGELOG.md
          git commit -m "chore(release): update changelog for ${{ github.ref_name }}"
          git push
```

### GitLab CI

```yaml
generate-changelog:
  image: rust:latest
  before_script:
    - cargo install git-cliff
  script:
    - git-cliff -o CHANGELOG.md
  artifacts:
    paths: [CHANGELOG.md]
  only:
    - tags
```

## Tool Selection Guide

| Situation | Recommended Tool |
|-----------|-----------------|
| Simple changelog generation | git-cliff |
| Full release pipeline | semantic-release |
| npm package release | standard-version |
| GitHub-only project | changelogithub |
| Monorepo with many packages | semantic-release with multi-semantic-release |
| Need custom formatting | git-cliff with custom templates |
| No Node.js in stack | git-cliff or keep-a-changelog (Python) |

## Changelog Best Practices

- **Always include version and date** — `## [1.2.0] - 2026-05-14`
- **Group by category** — Added, Changed, Fixed, Removed, BREAKING
- **Link to compare view** — `[1.2.0]: https://github.com/org/repo/compare/v1.1.0...v1.2.0`
- **No duplicate entries** — each change appears once
- **Human-readable descriptions** — not raw commit messages
- **Include PR/issue references** — `(#123)`
- **BREAKING CHANGES at top** — with migration notes
- **Versions in reverse chronological order** — newest first
