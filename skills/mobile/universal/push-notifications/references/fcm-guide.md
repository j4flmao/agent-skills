# FCM Guide

## HTTP v1 API

```
POST https://fcm.googleapis.com/v1/projects/{PROJECT_ID}/messages:send
Authorization: Bearer <OAUTH2_TOKEN>
Content-Type: application/json
```

### Obtain OAuth2 Token

```bash
# Service account JSON → JWT assertion → access token (60 min expiry)
gcloud auth application-default print-access-token
```

## Message Structure

```json
{
  "message": {
    "token": "device_token_or_topic",
    "notification": {
      "title": "Order Shipped",
      "body": "Your package is on its way"
    },
    "data": {
      "order_id": "12345",
      "type": "shipped"
    },
    "android": {
      "priority": "HIGH",
      "notification": {
        "channel_id": "orders",
        "sound": "default",
        "click_action": "OPEN_ORDER"
      }
    },
    "apns": {
      "headers": {
        "apns-priority": "10"
      },
      "payload": {
        "aps": {
          "sound": "default",
          "badge": 3,
          "mutable-content": 1
        }
      }
    }
  }
}
```

## Topic Messaging

```bash
# Subscribe
POST https://iid.googleapis.com/iid/v1/{TOKEN}/rel/topics/{TOPIC}
Authorization: key={SERVER_KEY}

# Publish — use HTTP v1 with "topic" field
```

## Device Group Messaging (Legacy)

```bash
POST https://fcm.googleapis.com/fcm/send
Content-Type: application/json
Authorization: key={SERVER_KEY}

{
  "to": "/topics/deals",
  "notification": { "title": "Flash Sale!" }
}
```

## Delivery Extensions

| Feature | Description |
|---------|-------------|
| `time_to_live` | Max 28 days; 0 = no store-and-forward |
| `priority` | `HIGH` wakes device; `NORMAL` waits for Doze maintenance |
| `collapse_key` | Replaces pending notification with same key |
| `direct_boot_ok` | Delivered before user unlocks device (Android 7+) |

## Token Lifecycle

- Generated by `FirebaseMessaging.getInstance().token`
- Token can change: app restore, new device, clear app data
- Listen with `onNewToken` callback; sync to server immediately

## Error Codes (HTTP v1)

| Code | Retry | Meaning |
|------|-------|---------|
| 200 | No | Success |
| 400 | No | Invalid JSON or missing fields |
| 401 | No | Auth token expired — refresh |
| 404 | No | Token unregistered — remove |
| 429 | Yes | Rate limited — exponential backoff |
| 500+ | Yes | Internal — wait then retry |
