# Developer Portal Design

## Portal Features
| Feature | Purpose |
|---------|---------|
| Interactive API reference | Try endpoints directly |
| API key management | Create, revoke, rotate keys |
| SDK download | Language-specific client libraries |
| Status page | Service health and incidents |
| Changelog | API changes and versions |
| Documentation | Guides, tutorials, best practices |
| Forum/community | Developer support and discussions |
| Usage dashboard | See your API consumption |

## Portal Platforms
| Platform | Best For |
|----------|----------|
| SwaggerHub | OpenAPI-first organizations |
| Stoplight | Design-first API workflow |
| ReadMe | Great developer experience out of box |
| Redoc | Beautiful API reference docs |
| Custom (Next.js) | Full control, branded experience |

## Self-Service Onboarding Flow
1. Developer visits portal
2. Signs up (email, GitHub, Google)
3. Gets API key immediately
4. Sees "Hello World" example in their language
5. Can test API in interactive playground
6. Documentation auto-generated from OpenAPI spec
