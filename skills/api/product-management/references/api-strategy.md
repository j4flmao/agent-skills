# API Product Strategy

## API Business Models
| Model | Description | Example |
|-------|-------------|---------|
| Free | No charge, drive ecosystem adoption | GitHub API |
| Usage-based | Pay per request/unit | Stripe, Twilio |
| Tiered | Different plans with limits | Google Maps API |
| Freemium | Free tier + paid premium | Slack API |
| Revenue share | Percentage of transaction | Stripe Connect |
| Internal value | Cost savings, not direct revenue | Platform APIs |

## API KPIs
| Metric | What It Measures | Target |
|--------|-----------------|--------|
| API adoption | Number of consumers/keys issued | Growth MoM |
| Time to first call | Time from signup to first successful call | < 5 min |
| API reliability | Uptime, error rate, p99 latency | 99.9% uptime |
| Developer NPS | Developer satisfaction | > 50 |
| API revenue | Direct/indirect API revenue | Target depends on model |
| Churn rate | % of consumers stopped using API | < 5%/month |

## API Governance
- API design guidelines (naming, pagination, error format)
- API review process (design-first approach)
- API versioning and deprecation policy
- API catalog for discoverability
- Rate limiting and throttling policies
